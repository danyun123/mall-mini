# 订单列表示例功能卡

## 目标

提供一个稳定的首个样板功能，用来示范业务列表页应如何按页面、store、service、request 分层方式落地。

## 本次范围

- 在订单分包内搭建订单列表页结构
- 定义获取订单列表所需的 request 和 service 契约
- 覆盖正常态、空态和错误态 UI
- 支持从列表跳转到后续的订单详情页

## 本次不做

- 高级筛选和排序
- 支付或结算流程
- 真实后端联调
- 详情页实现

## 数据预期

- 订单 ID
- 订单标题
- 供应商名称
- 金额摘要
- 订单状态
- 创建时间

## 交互预期

- 页面进入时自动首屏加载
- 支持下拉刷新
- 支持空态展示
- 支持错误兜底与重试
- 支持跳转到未来的详情页

## 验收标准

1. 页面实现遵守项目分层规则。
2. request、adapter、service 的职责边界清晰。
3. UI 覆盖正常态、空态和错误态。
4. 相关验证记录同步写入执行计划。

## 当前样板实现记录

- 已在 `miniprogram/pkg-order/pages/order-list` 新增订单列表样板页。
- 当前样板页使用本地 mock 数据，不接入真实后端接口。
- 业务类型、后端 DTO 适配和模拟场景编排都就近维护在页面目录下的 `service.ts`。
- 页面已覆盖正常态、空态和错误态，详情页暂以 toast 占位提示，不实现真实详情页。
- 验证记录见 `docs/exec-plans/order-list-sample.md`。
